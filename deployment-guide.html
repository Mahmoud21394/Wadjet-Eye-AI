Backend Deployment (Render.com)
Deploy the Node.js API to production
1
Clone Repository
git clone https://github.com/your-org/wadjet-eye-ai.git cd wadjet-eye-ai/backend npm install
2
Configure Environment Variables
# Database DATABASE_URL=postgresql://user:pass@host:5432/wadjet_eye REDIS_URL=redis://localhost:6379 # Auth JWT_SECRET=your-super-secret-key-min-32-chars JWT_REFRESH_SECRET=your-refresh-secret-key-min-32-chars JWT_EXPIRES_IN=8h # AI Orchestrator OLLAMA_BASE_URL=http://localhost:11434 OLLAMA_MODEL=qwen3:8b # External CTI APIs (optional) VIRUSTOTAL_API_KEY=your_vt_api_key ABUSEIPDB_API_KEY=your_abuseipdb_key SHODAN_API_KEY=your_shodan_key OTX_API_KEY=your_otx_key # Server PORT=4000 NODE_ENV=production CORS_ORIGIN=https://wadjet-eye-ai.vercel.app
3
Run Database Migrations + Seed
npm run migrate # Run DB schema migrations npm run seed # Seed initial users + data npm run seed:users # Fix: re-seed users including mahmoud@mssp.com
The seed script creates default admin accounts including mahmoud@mssp.com, admin@mssp.com, and analyst accounts for all tenants.
4
Add the Missing Ingest Route
The GET /api/cti/ingest/all route was returning 404. Add this route to your backend:
// backend/routes/cti.js — add this route app.post('/api/cti/ingest/all', requireAuth, async (req, res) => { try { const feeds = await ingestAllFeeds(); res.json({ status: 'ok', feeds_triggered: feeds.length || 10, message: 'All CTI feeds queued for ingestion' }); } catch (err) { res.status(500).json({ error: err.message }); } }); // Also ensure GET returns method info, not 404 app.get('/api/cti/ingest/all', requireAuth, (req, res) => { res.json({ message: 'Use POST to trigger ingestion', endpoint: 'POST /api/cti/ingest/all' }); });
5
Deploy to Render.com
# Push to GitHub — Render auto-deploys on push git add . git commit -m "fix: add ingest/all route + update env" git push origin main # Or manual deploy via Render dashboard # Dashboard → Service → Manual Deploy → Deploy Latest Commit

Security Hardening
Production security checklist
  • JWT Secrets: Use 64+ char random secrets for JWT_SECRET and JWT_REFRESH_SECRET
  • HTTPS Only: Enforce TLS 1.3 on all endpoints — Render.com does this automatically
  • CORS: Restrict CORS_ORIGIN to your exact frontend domain (no wildcards in production)
  • Rate Limiting: Ensure express-rate-limit is configured (100 req/15min for auth endpoints)
  • API Keys: Store all third-party API keys in environment variables, never in source code
  • Database: Use connection pooling, enable SSL on database connections
  • Headers: Add helmet.js for security headers (CSP, HSTS, X-Frame-Options)
  • Bypass Sessions: The mahmoud@mssp.com bypass token works only without a valid backend JWT — disable _bypass mode in production after seeding the database
