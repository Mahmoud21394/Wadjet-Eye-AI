1 Root Cause Analysis

🔴 Bug #1 — PAGE_CONFIG Race Condition (Critical)

File: js/campaigns-soc.js (v2.0), js/live-detections-soc.js (v2.0)

The SOC modules patched PAGE_CONFIG['campaigns'] inside a single setTimeout(fn, 150). If main.js executed its navigation before that timer fired, the original renderCampaigns() from pages.js (which uses ARGUS_DATA mock data) ran instead of the SOC module.

Fix: Both modules now install overrides at parse time (immediately) AND run a 40-attempt retry loop (every 150ms = 6 seconds total) to patch PAGE_CONFIG as soon as it's available. main.js itself was also updated to call renderCampaignsSOC() and renderLiveDetectionsSOC() directly if those functions are available.

🔴 Bug #2 — Mock Data Override Not Taking Effect

File: js/pages.js line 124 defines function renderCampaigns() as a hoisted function using ARGUS_DATA.campaigns (mock data). Because this is a regular function declaration, main.js's PAGE_CONFIG['campaigns'].onEnter called the local scope version, not the global one.

Fix: window.renderCampaigns is now overridden immediately at module parse time in campaigns-soc.js, and main.js PAGE_CONFIG now explicitly checks window.renderCampaignsSOC first.

🔴 Bug #3 — Double /api Prefix in API Calls

File: js/campaigns-soc.js, js/live-detections-soc.js, js/campaign-correlation.js (all v2.0)

The _apiGet(path) helper called window.authFetch('/api' + path) while authFetch in auth-interceptor.js already prepends base + '/api'. Result: every API call hit https://wadjet-eye-ai.onrender.com/api/api/cti/campaigns → 404 on every request.

Fix: All three files now pass path directly to window.authFetch(path) without the /api prefix. The direct fetch() fallback still manually adds /api.

🟠 Bug #4 — No Campaigns in Database (Auto-Correlation Not Running)

Even after the URL was fixed, the response returned an empty array because no campaigns had been created yet. The correlation engine was never triggered.

Fix: When the campaigns API returns 0 rows, the module now automatically invokes CorrelationEngine.check() after 3 seconds, which triggers ingestion + client-side IOC clustering to auto-create campaigns.

🟠 Bug #5 — stopDetections() ReferenceError on Page Leave

The onLeave for 'detections' called the local stopDetections() from live-pages.js which could be overridden or undefined.

Fix: live-detections-soc.js now overrides window.stopDetections with a safe wrapper, and main.js checks window.stopDetections first.

4 SOC Dashboard Architecture

Module Load Order (must be preserved)
index.html script tags (bottom of body):
  1. js/main.js               → defines PAGE_CONFIG['campaigns'], PAGE_CONFIG['detections']
  2. js/live-pages-patch.js   → patches renderCampaignsLive (old override)
  3. js/campaign-correlation.js → window.CorrelationEngine
  4. js/campaigns-soc.js      → window.renderCampaigns, window.renderCampaignsSOC (overrides #1)
  5. js/live-detections-soc.js → window.renderDetections, window.stopDetections (overrides #1)
Data Flow: IOC → Campaign → Detection → Case
Ingestion pipeline runs (CISA / OTX / AbuseIPDB / MISP)
  ↓ POST /api/ingest/run
  ↓ IOCs saved to iocs table with tenant_id, threat_actor, tags

Correlation engine runs (every 5 min)
  ↓ GET /api/iocs?limit=500&sort=risk_score&order=desc
  ↓ Clusters IOCs by: actor / malware_family / tags[0] / country+ASN
  ↓ POST /api/cti/campaigns for each cluster ≥ 3 IOCs

Campaigns page loads
  ↓ renderCampaignsSOC() called by PAGE_CONFIG['campaigns'].onEnter
  ↓ GET /api/cti/campaigns?page=1&limit=20
  ↓ Renders KPI cards + campaign table + IOC graph

Live detections page
  ↓ renderLiveDetectionsSOC() called by PAGE_CONFIG['detections'].onEnter
  ↓ GET /api/cti/detections + WebSocket /ws/detections
  ↓ Each event shows: severity, type, IOC, MITRE technique, campaign link

backend/routes/iocs.js — Service Role Fix

Ensure iocs.js uses the Supabase service-role client for server-side queries, bypassing RLS for the backend process:

// backend/routes/iocs.js — top of file
const { createClient } = require('@supabase/supabase-js');
const supabaseAdmin = createClient(
  process.env.SUPABASE_URL,
  process.env.SUPABASE_SERVICE_ROLE_KEY   // ← must be SERVICE_ROLE, not ANON
);

// In the GET /iocs handler:
const { tenant_id } = req.user || {};
let query = supabaseAdmin.from('iocs').select('*', { count: 'exact' });

// Only filter by tenant if NOT a super_admin
if (req.user?.role !== 'SUPER_ADMIN' && tenant_id) {
  query = query.eq('tenant_id', tenant_id);
}

8 Supabase SQL: Migration & RLS

Full migration-v7.0-ioc-rls.sql

-- ══════════════════════════════════════════════════════════
-- Wadjet-Eye AI — migration-v7.0-ioc-rls.sql
-- Run in Supabase SQL Editor
-- ══════════════════════════════════════════════════════════

-- 1. Enable RLS on iocs table
ALTER TABLE public.iocs ENABLE ROW LEVEL SECURITY;

-- 2. Drop old policies if they exist
DROP POLICY IF EXISTS "Enable read for authenticated users" ON public.iocs;
DROP POLICY IF EXISTS "tenant_select_iocs"   ON public.iocs;
DROP POLICY IF EXISTS "service_all_iocs"     ON public.iocs;
DROP POLICY IF EXISTS "super_admin_all_iocs" ON public.iocs;

-- 3. Tenant SELECT policy — users see only their tenant's IOCs
CREATE POLICY "tenant_select_iocs" ON public.iocs
FOR SELECT TO authenticated
USING (
  tenant_id = (auth.jwt() ->> 'tenant_id')
  OR (auth.jwt() ->> 'tenant_id') IS NULL  -- allow null tenant (legacy)
);

-- 4. SUPER_ADMIN sees all IOCs across all tenants
CREATE POLICY "super_admin_all_iocs" ON public.iocs
FOR ALL TO authenticated
USING (
  (auth.jwt() ->> 'role') IN ('SUPER_ADMIN', 'PLATFORM_ADMIN')
);

-- 5. Service role bypass (for backend iocs.js supabaseAdmin client)
CREATE POLICY "service_role_bypass" ON public.iocs
FOR ALL TO service_role
USING (true)
WITH CHECK (true);

-- 6. Ensure campaigns table has RLS + tenant policy
ALTER TABLE public.campaigns ENABLE ROW LEVEL SECURITY;
DROP POLICY IF EXISTS "tenant_campaigns" ON public.campaigns;
CREATE POLICY "tenant_campaigns" ON public.campaigns
FOR SELECT TO authenticated
USING (
  tenant_id = (auth.jwt() ->> 'tenant_id')
  OR (auth.jwt() ->> 'role') IN ('SUPER_ADMIN', 'PLATFORM_ADMIN')
);

-- 7. Verify
SELECT tablename, policyname, cmd, qual
FROM pg_policies
WHERE tablename IN ('iocs', 'campaigns')
ORDER BY tablename, policyname;
