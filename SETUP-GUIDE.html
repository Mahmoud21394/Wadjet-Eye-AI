🔧 Troubleshooting Common Errors

Error: ❌ SEED ERROR: Missing required environment variables: SUPABASE_URL, SUPABASE_SERVICE_KEY
Fix: Your backend/.env file is missing or has wrong variable names. Check the file exists at backend/.env (not .env.example). Run: cat backend/.env and verify both keys are present.
Error: Cannot find module '@supabase/supabase-js'
Fix: Dependencies not installed.
PS>cd backend && npm install
Error: Table "tenants" does not exist
Fix: Schema not applied to Supabase yet. Go to Supabase SQL Editor and run backend/database/schema.sql first (Step 1 above).
Error: Auth creation failed: User already registered
Fix: The Supabase Auth user already exists — this is fine. The script will update the password and link the profile. The seed is safe to re-run.
Error: ❌ User profile not found. Contact your administrator. (when logging in)
Fix: The Supabase Auth user was created but the users profile row was not inserted (or auth_id column is NULL).
  1. Re-run the seed: node backend/scripts/seed-mahmoud.js
  2. Check Supabase → Table Editor → users table — look for the row with your email
  3. If the row exists but auth_id is NULL — run the SQL fix below
-- Run in Supabase SQL Editor to link auth_id manually
UPDATE public.users
SET auth_id = (
  SELECT id FROM auth.users WHERE email = 'mahmoud@mssp.com' LIMIT 1
)
WHERE email = 'mahmoud@mssp.com' AND auth_id IS NULL;
Error: relation "refresh_tokens" does not exist (backend crash)
Fix: Run migration-v5.0-auth-tables.sql in Supabase SQL Editor (Step 2 above).
Error: duplicate key value violates unique constraint (during seed)
Fix: Normal — data already exists. The seed uses upsert and ON CONFLICT DO NOTHING, so this is safe. No action needed.
