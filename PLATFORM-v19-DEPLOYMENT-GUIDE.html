VirusTotal API Key→ virustotal.com/gui/my-apikey (Free: 4 req/min)

How to Configure Keys

Method 1 — Via Platform Settings UI:
Navigate to Settings → AI Configuration (for OpenAI/Claude) and Settings → Threat Intelligence Feeds (for VT/AbuseIPDB/Shodan/OTX)

Method 2 — Via AI Orchestrator:
Click API Keys button in the AI Orchestrator header → fill in keys → Save

Method 3 — Backend Environment Variables:

🔧 Platform Settings HTTP 400 Fix (v3.0)

✅ Root Cause Fixed: The HTTP 400 error was caused by sending null/empty/undefined fields in the JSON payload, which failed backend validation.

What Was Fixed

The new _collectSettings() function:

  • ✅ Strips all null, undefined, and empty string "" values before sending
  • ✅ Preserves boolean false and number 0 values (these are valid)
  • ✅ Implements a 3-tier fallback save chain: PUT /settings → PATCH /settings → PATCH /settings/{section}
  • ✅ Offline edit mode: if all API calls fail, settings are cached to localStorage
  • ✅ API keys from settings are synced to localStorage for use by AI Orchestrator and IOC Lookup

Settings API Endpoints (Backend)

MethodEndpointDescription
GET/api/settingsLoad all settings (returns merged defaults + DB values)
PUT/api/settingsFull settings replacement (preferred)
PATCH/api/settingsPartial settings update (fallback)
PATCH/api/settings/{section}Update single section (section-level fallback)
⚠️ Ensure your backend GET /api/settings returns merged defaults so the frontend never receives null for required fields. See backend/routes/settings.js for the reference implementation.

CORS Configuration (Required for Client-Side API Calls)

For VirusTotal, AbuseIPDB, Shodan, OTX API calls to work from the browser, these external services must allow CORS. Most have CORS enabled for their v3 APIs. If blocked, proxy the requests through your backend.

# Express.js CORS — backend/server.js app.use(cors({ origin: ['https://your-frontend.com', 'http://localhost:3000'], credentials: true, methods: ['GET','POST','PUT','PATCH','DELETE','OPTIONS'], allowedHeaders: ['Content-Type','Authorization','x-api-key'], }));
