6.2 Why RLS Returns Empty Arrays (Explained)

Supabase Row-Level Security works like a WHERE clause appended to every query. When RLS is enabled but no SELECT policy exists, Supabase uses default-deny: every row is filtered out. The query succeeds (HTTP 200) but returns { data: [], count: 0 }. This is intentional behavior — it prevents accidental data exposure — but it's silent and misleading.

-- What the anon client effectively executes (simplified):
SELECT * FROM iocs
WHERE tenant_id = '00000000-0000-0000-0000-000000000001'
  AND FALSE  -- injected by RLS default-deny when no policy matches
LIMIT 50;
-- Result: 0 rows (even though 33,750 rows match the tenant filter)

6.4 Production RLS Policy (tenant-scoped)

-- Step 1: Remove debug policy
DROP POLICY IF EXISTS "debug_iocs_all_authenticated" ON iocs;

-- Step 2: Create tenant-scoped policies
-- Option A: Based on user's tenant_id in public.users
CREATE POLICY "iocs_tenant_read"
ON iocs FOR SELECT
TO authenticated
USING (
  tenant_id = (
    SELECT tenant_id FROM public.users
    WHERE auth_id = auth.uid()
    LIMIT 1
  )
);

-- Option B: Service role always bypasses (for backend reads)
CREATE POLICY "iocs_service_role_all"
ON iocs FOR ALL
TO service_role
USING (true)
WITH CHECK (true);

-- Option C: SUPER_ADMIN users can see all tenants
CREATE POLICY "iocs_super_admin_read"
ON iocs FOR SELECT
TO authenticated
USING (
  EXISTS (
    SELECT 1 FROM public.users
    WHERE auth_id = auth.uid()
    AND role IN ('SUPER_ADMIN', 'ADMIN')
  )
);

-- Apply all three for production:
ALTER TABLE iocs ENABLE ROW LEVEL SECURITY;
-- Then run the three CREATE POLICY statements above.

6.7 Scaling Best Practices (Threat Intel Platforms)

  • Partition by tenant_id — for >1M rows: PARTITION BY HASH(tenant_id) keeps per-tenant queries on single partitions
  • Archive stale IOCs — move IOCs older than 6 months to iocs_archive; active table stays <100K rows
  • Connection pooling — use Supabase Pooler (PgBouncer) for backend; set pool_mode=transaction
  • Rate-limit enrichment — VT API is 4 req/min on free; use enrichment queue (ioc_enrichment_queue table) with worker
  • Cache stats — cache /api/ingest/stats for 60s (Redis or in-memory); prevents 5 COUNT queries per page load
  • Deduplicate on ingest — the UNIQUE(tenant_id, value) constraint prevents duplicates; upsert strategy preserves highest risk_score
  • VACUUM schedule — IOC tables with frequent upserts need aggressive autovacuum; set autovacuum_vacuum_scale_factor = 0.05

§7 Security-Validation Checklist

7.1 XSS Prevention

  • IOC value rendering — all IOC values pass through _esc() (HTML entity encoding) before innerHTML insertion. Verify: inject <img src=x onerror=alert(1)> as an IOC value. Expected: rendered as text, no alert fires.
  • Search input — search box uses oninput → iocdbOnSearch() → sanitizeSearch(). Verify: type ";<script>alert(1)</script> in search box. Expected: special chars stripped, no XSS.
  • Error messages — error messages use _esc(err.message). Verify: cause a server error with a malicious message. Expected: HTML entities shown, no script execution.
  • Modal content — IOC detail modal renders via template literals with _esc(). Verify: create IOC with value='<b>bold</b>'. Expected: shown literally, not rendered as HTML.
  • CSP header — verify Content-Security-Policy header on backend responses restricts script-src to self + CDN only.

7.2 Token Theft Simulation

  • localStorage access — open DevTools > Console > localStorage.getItem('wadjet_access_token'). Note: v6.0 goal is httpOnly cookies; v6.1 bridge still uses localStorage for API compat. Risk: low with short 15-min TTL.
  • Token TTL enforcement — decode the JWT: atob(token.split('.')[1]). Verify exp is within 15 minutes of iat.
  • Token reuse after logout — logout, then try fetch('/api/iocs', { headers: { Authorization: 'Bearer ' + oldToken }}). Expected: 401 (token revoked or expired).
  • Refresh token rotation — verify each /api/auth/refresh call returns a NEW refresh token and the old one is invalidated (check refresh_tokens table: old token should be marked used=true).
  • Cross-tenant token test — log in as tenant 001 user, manually change X-Tenant-ID header to 002. Expected: still sees only tenant 001 data (tenant_id enforced in query, not header).

7.3 MFA Bypass Tests

  • Skip MFA challenge — intercept login response with mfa_required: true. Manually call _finalizeLogin(userData, null). Expected: _finalizeLogin requires a valid mfa_session_token; null token fails at /api/auth/mfa/challenge.
  • Replay TOTP code — use the same 6-digit TOTP code twice. Expected: second use returns { error: "TOTP code already used" } (code marked used in mfa_challenges table).
  • Expired challenge — wait for MFA session token to expire (5 min). Try to verify. Expected: { error: "MFA session expired" }.
  • Brute-force TOTP — send 10 wrong TOTP codes in rapid succession. Expected: rate limiting kicks in (429) after 5 attempts.
  • MFA enforcement for admins — verify ADMIN/SUPER_ADMIN accounts have mfa_enforced=true in DB. Attempt login without MFA enrollment. Expected: { error: "MFA required — please enroll first" }.

7.4 SQL / PostgREST Injection

  • Search injection — search for ',tenant_id.eq.00000000-0000-0000-0000-000000000002. Expected: commas/quotes stripped by sanitizeSearch(), no cross-tenant data.
  • Sort injection — send sort=risk_score;DROP TABLE iocs. Expected: validated against SORTABLE_COLS whitelist, falls back to risk_score, no SQL executed.
  • Pivot field injection — send POST /api/iocs/pivot with field='id;DROP TABLE iocs'. Expected: validated against validFields whitelist, returns 400.

7.5 RLS Bypass Tests

  • Anon access — call GET /api/iocs without Authorization header. Expected: 401 MISSING_TOKEN.
  • Other tenant's IOC by ID — get an IOC ID from tenant 002 (from admin). Try GET /api/iocs/<tenant-002-ioc-id> as tenant 001 user. Expected: 404 (query includes .eq('tenant_id', tenant001Id)).
  • Stats cross-tenant — as ANALYST, verify /api/ingest/stats returns only tenant 001 counts, not all-tenant totals (all-tenant fallback is ADMIN-only).
